# Storage Directory

This directory contains all data generated by the intrusion detection system.

## Directory Structure

```
storage/
├── db/              # SQLite database files
│   └── events.sqlite    # Main database (auto-created)
├── videos/          # Full video recordings
├── snaps/           # Snapshot images from detections
└── clips/           # Short video clips of events
```

## Database

- **Type:** SQLite (file-based, no server required)
- **Location:** `db/events.sqlite`
- **Contents:** Events, zones, models, users, SOS logs
- **Backup:** Simply copy the .sqlite file

## Videos

- **Full recordings** of camera feeds
- **Automatic cleanup** based on storage settings
- **Formats:** MP4, AVI, MOV (depends on camera)

## Snapshots

- **Images captured** during detection events  
- **JPEG format** for efficient storage
- **Automatic naming** with timestamps

## Clips

- **Short video segments** around detection events
- **Configurable length** (e.g., 10 seconds before/after)
- **Linked to events** in database

## Storage Management

- **Automatic cleanup** when limits are reached
- **Configurable retention** periods
- **Manual cleanup** via GUI Settings tab
- **Size monitoring** in System Status tab

## Backup Recommendations

1. **Regular database backup:** Copy `db/events.sqlite`
2. **Important events:** Export via GUI
3. **Critical videos:** Copy to external storage
4. **Zone configurations:** Export via Zone Editor

## Privacy Note

All data is stored locally on your system. No data is sent to external services unless explicitly configured.
